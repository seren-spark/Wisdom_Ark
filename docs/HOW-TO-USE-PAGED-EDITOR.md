# 🚀 分页编辑器使用指南

## 快速开始

分页编辑器已经完全集成到项目中，你可以通过以下步骤使用：

---

## 📋 使用步骤

### 1. 加载测试文档

点击右上角的 **实验** 图标（三个点），选择：

- **大文档 (5万字)** - 推荐用于测试分页功能

### 2. 转换为分页格式

文档加载完成后，再次点击右上角菜单，选择：

- **📄 转换为分页格式**

系统会：

- 按每页 5000 字拆分文档
- 提取所有标题构建目录索引
- 保存到 IndexedDB

完成后会显示统计信息：

```
✅ 转换完成，耗时 XXms
📊 文档已拆分为 X 页，共 XX 字，XX 个标题
💡 点击"启用分页模式"按钮以使用分页编辑器
```

### 3. 启用分页模式

点击菜单中的：

- **🚀 启用分页模式**

系统会：

- 创建分页管理器
- 只加载前 3 页到编辑器
- 开始滚动监听

你会看到：

```
✅ 分页模式已启用
```

### 4. 体验分页编辑

**滚动测试：**

- 向下滚动，系统会自动加载新页面
- 向上滚动，远离的页面会被卸载
- 控制台会显示详细日志

**编辑测试：**

- 正常编辑内容
- 2 秒后自动保存
- 离开页面时会先保存再卸载

**目录跳转：**

- 点击左侧目录
- 自动加载目标页并跳转

### 5. 恢复完整文档

点击菜单中的：

- **🔙 恢复完整文档**

系统会：

- 从 IndexedDB 加载所有页面
- 合并为完整文档
- 恢复到编辑器

---

## 🎯 功能说明

### 内存优化

| 模式         | DOM 节点数 | 内存占用 |
| ------------ | ---------- | -------- |
| **普通模式** | ~2000+     | ~100MB   |
| **分页模式** | ~100       | ~30MB    |

### 自动保存

- 编辑后 2 秒自动保存
- 卸载页面前强制保存
- 防止数据丢失

### 边界保护

- 编辑中的页面不会被卸载
- 选区跨页时不会卸载
- 确保编辑稳定性

## 🐛 调试

### 查看控制台日志

打开浏览器控制台，可以看到详细日志：

```
[PagedEditor] 初始化，起始页: 1
[PagedEditor] 加载 Page 1 成功，插入位置: 0
[PagedEditor] 切换到 Page 2
[PagedEditor] 卸载 Page 1 成功
[PagedEditor] 保存 Page 2 成功，字数: 4523
```

### 查看 IndexedDB 数据

1. 打开浏览器开发者工具
2. Application → Storage → IndexedDB
3. 找到 `PagedDocuments` 数据库
4. 查看：
   - `documents` - 文档元数据
   - `pages` - 页面内容
   - `tocIndexes` - 目录索引

### 查看统计信息

在控制台执行：

```javascript
import { DocumentConverter } from '@/pages/editor/utils/DocumentConverter';

const stats = await DocumentConverter.getDocumentStats('current-doc');
console.log(stats);
```

输出：

```json
{
  "totalPages": 10,
  "totalWords": 50000,
  "totalHeadings": 45,
  "averagePageSize": 5000
}
```

## ⚠️ 注意事项

### ✅ 适用场景

- **文档字数 > 10,000 字** - 分页收益明显
- **文档字数 > 50,000 字** - 强烈推荐

### ❌ 不适用场景

- **文档字数 < 3,000 字** - 没必要分页
- **需要频繁全文搜索** - 只能搜索已加载的页面

### 📌 使用建议

1. **先转换，后启用**
   - 不要跳过"转换为分页格式"步骤
   - 转换后数据会持久化保存

2. **编辑时注意**
   - 编辑会自动保存，无需手动保存
   - 离开页面前会自动保存

3. **性能监控**
   - 按 `Ctrl+Shift+P` 打开性能监控
   - 观察 DOM 节点数变化

## 🔧 配置选项

### 修改页面大小

编辑 `src/pages/editor/draft.tsx`：

```typescript
await DocumentConverter.convertToPagedDocument(
  editor,
  pagedDocId,
  '当前文档',
  3000, // 改为 3000 字/页（默认 5000）
);
```

### 修改保存延迟

编辑 `src/pages/editor/managers/PagedEditorManager.ts`：

```typescript
this.autoSaveTimer = window.setTimeout(() => {
  this.autoSave();
}, 1000); // 改为 1 秒（默认 2 秒）
```
