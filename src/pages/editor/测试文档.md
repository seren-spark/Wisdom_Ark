# React 性能优化完全指南

## 1. 组件渲染优化

### 1.1 使用 React.memo 避免不必要的重渲染

React.memo 是一个高阶组件（HOC），它通过对 props 进行浅比较来决定是否需要重新渲染组件。当组件的 props 没有变化时，React.memo 会跳过渲染，直接复用最近一次渲染的结果。

使用场景：

- 纯展示组件，props 变化频率低
- 列表项组件，避免整个列表重渲染
- 复杂计算的组件，减少不必要的计算

代码示例：

```jsx
import React from 'react';

function UserCard({ name, age, avatar }) {
  console.log('UserCard 渲染了');
  return (
    <div className="user-card">
      <img src={avatar} alt={name} />
      <h3>{name}</h3>
      <p>年龄: {age}</p>
    </div>
  );
}

const MemoizedUserCard = React.memo(UserCard);

const MemoizedUserCardWithCustomCompare = React.memo(UserCard, (prevProps, nextProps) => {
  return prevProps.name === nextProps.name && prevProps.age === nextProps.age;
});
```

注意事项：

1. React.memo 只进行浅比较，对于复杂对象需要自定义比较函数
2. 不要过度使用，简单组件的比较成本可能高于重渲染成本
3. 配合 useMemo 和 useCallback 使用效果更好

### 1.2 使用 useMemo 缓存计算结果

useMemo 用于缓存计算开销较大的结果，只有当依赖项发生变化时才会重新计算。

使用场景：

- 复杂的数据处理和计算
- 数组的过滤、排序、映射操作
- 需要传递给子组件的对象或数组

代码示例：

```jsx
import React, { useMemo, useState } from 'react';

function ProductList({ products, category }) {
  const [sortBy, setSortBy] = useState('price');

  const filteredAndSortedProducts = useMemo(() => {
    console.log('重新计算产品列表');
    const filtered = products.filter((p) => p.category === category);
    return filtered.sort((a, b) => {
      if (sortBy === 'price') return a.price - b.price;
      if (sortBy === 'name') return a.name.localeCompare(b.name);
      return 0;
    });
  }, [products, category, sortBy]);

  return (
    <div>
      <select value={sortBy} onChange={(e) => setSortBy(e.target.value)}>
        <option value="price">按价格排序</option>
        <option value="name">按名称排序</option>
      </select>
      <ul>
        {filteredAndSortedProducts.map((product) => (
          <li key={product.id}>
            {product.name} - ¥{product.price}
          </li>
        ))}
      </ul>
    </div>
  );
}
```

性能对比：

- 不使用 useMemo：每次渲染都会重新计算，1000 个商品约 5-10ms
- 使用 useMemo：只在依赖变化时计算，其他时候小于 0.1ms

### 1.3 使用 useCallback 缓存函数引用

useCallback 用于缓存函数引用，避免每次渲染都创建新的函数实例，特别是当函数作为 props 传递给子组件时。

使用场景：

- 传递给使用了 React.memo 的子组件的回调函数
- 作为 useEffect 的依赖项的函数
- 传递给第三方库的回调函数

代码示例：

```jsx
import React, { useState, useCallback } from 'react';

function TodoList() {
  const [todos, setTodos] = useState([]);
  const [filter, setFilter] = useState('all');

  const handleAddTodo = useCallback((text) => {
    setTodos((prev) => [...prev, { id: Date.now(), text, completed: false }]);
  }, []);

  const handleToggleTodo = useCallback((id) => {
    setTodos((prev) =>
      prev.map((todo) => (todo.id === id ? { ...todo, completed: !todo.completed } : todo)),
    );
  }, []);

  const handleDeleteTodo = useCallback((id) => {
    setTodos((prev) => prev.filter((todo) => todo.id !== id));
  }, []);

  return (
    <div>
      <TodoInput onAdd={handleAddTodo} />
      <FilterButtons filter={filter} onChange={setFilter} />
      <TodoItems todos={todos} onToggle={handleToggleTodo} onDelete={handleDeleteTodo} />
    </div>
  );
}

const TodoInput = React.memo(({ onAdd }) => {
  const [text, setText] = useState('');
  const handleSubmit = (e) => {
    e.preventDefault();
    if (text.trim()) {
      onAdd(text);
      setText('');
    }
  };
  return (
    <form onSubmit={handleSubmit}>
      <input value={text} onChange={(e) => setText(e.target.value)} />
      <button type="submit">添加</button>
    </form>
  );
});
```

## 2. 列表渲染优化

### 2.1 虚拟化长列表

对于包含大量数据的列表（如 1000+ 条），普通渲染会导致严重的性能问题。虚拟化技术只渲染可见区域的元素，大幅提升性能。

推荐库：

- react-window：轻量级，适合简单场景
- react-virtualized：功能丰富，适合复杂场景
- @tanstack/react-virtual：现代化，TypeScript 友好

react-window 示例：

```jsx
import React from 'react';
import { FixedSizeList } from 'react-window';

function VirtualizedList({ items }) {
  const Row = ({ index, style }) => (
    <div style={style} className="list-item">
      <h4>{items[index].title}</h4>
      <p>{items[index].description}</p>
    </div>
  );

  return (
    <FixedSizeList height={600} itemCount={items.length} itemSize={80} width="100%">
      {Row}
    </FixedSizeList>
  );
}
```

性能对比：

- 普通渲染 10000 条：首次渲染 2000ms，滚动卡顿
- 虚拟化渲染 10000 条：首次渲染 50ms，滚动流畅 60fps

### 2.2 使用 key 优化列表更新

正确使用 key 可以帮助 React 识别哪些元素发生了变化，避免不必要的 DOM 操作。

错误示例：

```jsx
{
  items.map((item, index) => <TodoItem key={index} todo={item} />);
}

{
  items.map((item) => <TodoItem key={Math.random()} todo={item} />);
}
```

正确示例：

```jsx
{
  items.map((item) => <TodoItem key={item.id} todo={item} />);
}

{
  items.map((item) => <TodoItem key={`${item.userId}-${item.timestamp}`} todo={item} />);
}
```

## 3. 代码分割与懒加载

### 3.1 使用 React.lazy 和 Suspense

代码分割可以将应用拆分成多个小块，按需加载，减少首屏加载时间。

基础用法：

```jsx
import React, { Suspense, lazy } from 'react';

const Dashboard = lazy(() => import('./pages/Dashboard'));
const Profile = lazy(() => import('./pages/Profile'));
const Settings = lazy(() => import('./pages/Settings'));

function App() {
  return (
    <div>
      <Suspense fallback={<div>加载中...</div>}>
        <Routes>
          <Route path="/dashboard" element={<Dashboard />} />
          <Route path="/profile" element={<Profile />} />
          <Route path="/settings" element={<Settings />} />
        </Routes>
      </Suspense>
    </div>
  );
}
```

高级用法：预加载

```jsx
import React, { Suspense, lazy } from 'react';

const HeavyComponent = lazy(() => import('./HeavyComponent'));

function App() {
  const handleMouseEnter = () => {
    import('./HeavyComponent');
  };

  return (
    <div>
      <button onMouseEnter={handleMouseEnter}>打开重型组件</button>
      <Suspense fallback={<Loading />}>
        <HeavyComponent />
      </Suspense>
    </div>
  );
}
```

### 3.2 路由级别的代码分割

在路由层面进行代码分割是最常见的优化方式。

React Router 示例：

```jsx
import { lazy, Suspense } from 'react';
import { BrowserRouter, Routes, Route } from 'react-router-dom';

const Home = lazy(() => import('./pages/Home'));
const About = lazy(() => import('./pages/About'));
const Contact = lazy(() => import('./pages/Contact'));
const Admin = lazy(() => import('./pages/Admin'));

function App() {
  return (
    <BrowserRouter>
      <Suspense fallback={<PageLoader />}>
        <Routes>
          <Route path="/" element={<Home />} />
          <Route path="/about" element={<About />} />
          <Route path="/contact" element={<Contact />} />
          <Route path="/admin/*" element={<Admin />} />
        </Routes>
      </Suspense>
    </BrowserRouter>
  );
}
```

效果：

- 首屏 bundle 大小：从 500KB 降到 150KB
- 首屏加载时间：从 3s 降到 1s
- 其他页面按需加载，用户体验更好

## 4. 状态管理优化

### 4.1 避免不必要的全局状态

不是所有状态都需要放在全局状态管理中，合理使用本地状态可以减少不必要的渲染。

状态分类：

- 本地状态：表单输入、UI 交互状态（折叠/展开）
- 共享状态：用户信息、主题设置
- 服务端状态：API 数据（推荐使用 React Query）

### 4.2 使用 Context 时的性能陷阱

Context 的值变化会导致所有消费者重新渲染，需要合理拆分。

错误示例：

```jsx
const AppContext = createContext();

function App() {
  const [user, setUser] = useState(null);
  const [theme, setTheme] = useState('light');
  const [language, setLanguage] = useState('zh');

  const value = { user, setUser, theme, setTheme, language, setLanguage };

  return (
    <AppContext.Provider value={value}>
      <AppContent />
    </AppContext.Provider>
  );
}
```

正确示例：拆分 Context

```jsx
const UserContext = createContext();
const ThemeContext = createContext();
const LanguageContext = createContext();

function App() {
  const [user, setUser] = useState(null);
  const [theme, setTheme] = useState('light');
  const [language, setLanguage] = useState('zh');

  return (
    <UserContext.Provider value={{ user, setUser }}>
      <ThemeContext.Provider value={{ theme, setTheme }}>
        <LanguageContext.Provider value={{ language, setLanguage }}>
          <AppContent />
        </LanguageContext.Provider>
      </ThemeContext.Provider>
    </UserContext.Provider>
  );
}
```

## 5. 图片和资源优化

### 5.1 图片懒加载

使用 Intersection Observer API 实现图片懒加载。

```jsx
import React, { useEffect, useRef, useState } from 'react';

function LazyImage({ src, alt, placeholder }) {
  const [imageSrc, setImageSrc] = useState(placeholder);
  const [isLoaded, setIsLoaded] = useState(false);
  const imgRef = useRef();

  useEffect(() => {
    const observer = new IntersectionObserver(
      (entries) => {
        entries.forEach((entry) => {
          if (entry.isIntersecting) {
            setImageSrc(src);
            observer.unobserve(entry.target);
          }
        });
      },
      { rootMargin: '50px' },
    );

    if (imgRef.current) {
      observer.observe(imgRef.current);
    }

    return () => observer.disconnect();
  }, [src]);

  return (
    <img
      ref={imgRef}
      src={imageSrc}
      alt={alt}
      onLoad={() => setIsLoaded(true)}
      style={{ opacity: isLoaded ? 1 : 0.5, transition: 'opacity 0.3s' }}
    />
  );
}
```

## 6. 性能监控和分析

使用步骤：

4. 操作应用
5. 停止录制，查看火焰图

### 6.2 使用 Performance API

```jsx
import { useEffect } from 'react';

function PerformanceMonitor({ componentName }) {
  useEffect(() => {
    const startTime = performance.now();

    return () => {
      const endTime = performance.now();
      const renderTime = endTime - startTime;

      if (renderTime > 16) {
        console.warn(`${componentName} 渲染耗时: ${renderTime.toFixed(2)}ms`);
      }
    };
  });

  return null;
}
```

## 7. 网络请求优化

### 7.1 使用 React Query 管理服务端状态

React Query 提供了强大的缓存、重试、预取等功能。

```jsx
import { useQuery, useMutation, useQueryClient } from '@tanstack/react-query';

function UserProfile({ userId }) {
  const queryClient = useQueryClient();

  const { data, isLoading, error } = useQuery({
    queryKey: ['user', userId],
    queryFn: () => fetchUser(userId),
    staleTime: 5 * 60 * 1000,
    cacheTime: 10 * 60 * 1000,
  });

  const updateMutation = useMutation({
    mutationFn: updateUser,
    onSuccess: () => {
      queryClient.invalidateQueries(['user', userId]);
    },
  });

  if (isLoading) return <div>加载中...</div>;
  if (error) return <div>错误: {error.message}</div>;

  return (
    <div>
      <h2>{data.name}</h2>
      <button onClick={() => updateMutation.mutate({ userId, name: '新名字' })}>更新</button>
    </div>
  );
}
```

### 7.2 请求去重和防抖

避免短时间内发送重复请求。

```jsx
import { useState, useCallback } from 'react';
import { debounce } from 'lodash';

function SearchBox() {
  const [results, setResults] = useState([]);

  const searchAPI = useCallback(
    debounce(async (query) => {
      if (!query) return;
      const data = await fetch(`/api/search?q=${query}`).then((r) => r.json());
      setResults(data);
    }, 300),
    [],
  );

  return (
    <div>
      <input onChange={(e) => searchAPI(e.target.value)} />
      <ul>
        {results.map((item) => (
          <li key={item.id}>{item.title}</li>
        ))}
      </ul>
    </div>
  );
}
```

## 9. 渲染优化技巧

### 9.1 避免在渲染中创建新对象

每次渲染创建新对象会导致子组件不必要的重渲染。

错误示例：

```jsx
function Parent() {
  return <Child style={{ color: 'red' }} />;
}
```

正确示例：

```jsx
const childStyle = { color: 'red' };

function Parent() {
  return <Child style={childStyle} />;
}
```

### 9.2 使用 CSS 而非 JavaScript 动画

CSS 动画由浏览器优化，性能更好。

```css
.fade-in {
  animation: fadeIn 0.3s ease-in;
}

@keyframes fadeIn {
  from {
    opacity: 0;
  }
  to {
    opacity: 1;
  }
}
```

### 9.3 使用 Web Workers 处理复杂计算

将耗时计算移到 Web Worker，避免阻塞主线程。

```jsx
import { useEffect, useState } from 'react';

function HeavyComputation({ data }) {
  const [result, setResult] = useState(null);

  useEffect(() => {
    const worker = new Worker('/worker.js');

    worker.postMessage(data);

    worker.onmessage = (e) => {
      setResult(e.data);
    };

    return () => worker.terminate();
  }, [data]);

  return <div>{result}</div>;
}
```

## 10. 总结

React 性能优化的核心原则：

1. 减少不必要的渲染
   - 使用 React.memo 包裹纯组件
   - 使用 useMemo 缓存计算结果
   - 使用 useCallback 缓存函数引用

2. 优化列表渲染
   - 虚拟化长列表（react-window）
   - 正确使用 key 属性
   - 避免在渲染中创建新对象

3. 代码分割和懒加载
   - 使用 React.lazy 和 Suspense
   - 路由级别的代码分割
   - 预加载关键资源

4. 合理的状态管理
   - 避免过度使用全局状态
   - 拆分 Context 避免不必要的渲染
   - 使用 React Query 管理服务端状态

5. 资源优化
   - 图片懒加载和 WebP 格式
   - 使用 CDN 加速静态资源
   - Tree Shaking 和代码压缩

6. 持续监控和分析
   - 使用 React DevTools Profiler
   - 使用 Performance API 监控性能
   - 建立性能预算和监控体系

记住：过早优化是万恶之源。先让代码工作，再用工具找出真正的性能瓶颈，然后针对性优化。不要为了优化而优化，要基于数据和用户体验做决策。

性能优化是一个持续的过程，需要在开发过程中不断关注和改进。通过合理使用 React 提供的优化工具和遵循最佳实践，可以构建出高性能的 React 应用。
